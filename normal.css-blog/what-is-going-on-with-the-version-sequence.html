Version sequence "bizarreness".

You might be asking why is it bizarre when it comes to version sequence

Semantic versioning talk

https://semver.org/ says MAJOR.MINOR.PATCH as semantic versioning, while this one is MAJOR.MINOR-OR-PATCH. This is standard to only most software, and as a developer of a project, you decide everything you put in, including source code releases, version sequences, and everything.

And what about Normal.css?

Well, we turn to customizing. We crafted the version sequence MAJOR.MINOR-OR-PATCH in an effort to separation. As mentioned earlier, customization is a priority to software that wants to spread out from the most-used things. And, we decided to go this way.

Conclusion and answer

We decided to go much quickly to 1.0, just so that we could simply enter the market of stable quick.

But what about sharing that to public?

Yes, we call this Fast-to-1 versioning (not trademarked). However it is not software, so we're not making a repo (lol).
